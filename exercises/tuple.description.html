In the following exercises, it will be useful to test our code by evaluating multiple expressions at once. This can be done using tuples, which are fixed-length collections of values that don't necessarily all have the same type as one another. Tuples can be defined as follows.

  1. The empty tuple is \s.s
  2. The 1-tuple (x) is \s.s x
  3. The 2-tuple (x, y) is \s.s x y
  4. The 3-tuple (x, y, z) is \s.s x y z

And so on.

Try evaluating a few tuples. Note that you can get friendlier output by adding the comment {:: tuple[TYPE1, TYPE2, ...] } (e.g., {:: tuple[expr, expr] }) above your expression.
